GeoDownloader 技术架构与特色
==============================

一、技术栈

后端（Rust）：
- Tauri 2.x 框架：轻量级跨平台桌面框架，比 Electron 体积小 10 倍以上
- tokio 异步运行时：高性能并发下载
- reqwest：HTTP 客户端，支持代理、自定义请求头
- image：图像处理（解码、合成、裁剪）
- rayon：CPU 并行计算（多核瓦片解码）
- rusqlite：本地 SQLite 数据库（缓存、MBTiles、GeoPackage）

前端（TypeScript + React）：
- React 19：最新版本，支持 Suspense 和并发特性
- Vite：快速构建工具
- shadcn/ui：高质量 UI 组件库
- Tailwind CSS v4：实用优先的样式方案
- Leaflet：2D 地图展示
- Cesium：3D 场景展示
- Zustand：轻量状态管理

二、性能特色

流式导出引擎（v3.5.0+）：
- 内存使用恒定：逐行处理，不受区域大小影响
- 四阶段并行流水线：下载→解码→编码→写入同时进行
- 多核加速：rayon 并行解码，8 核 CPU 可获得 4-6 倍加速
- 支持 BigTIFF：超过 4GB 的大文件自动切换 BigTIFF 格式

智能缓存系统：
- LRU 淘汰：按容量限制自动清理最久未使用的缓存
- 批量查询优化：1258 个瓦片缓存检查仅需 50ms（v3.4.5 优化前需 10 秒）
- 零拷贝共享：缓存数据通过内存引用复用，无需复制到临时文件
- 多数据源隔离：每个数据源独立命名空间

下载引擎：
- 最高 100 并发连接
- User-Agent 轮换：多种浏览器标识避免封锁
- 自动重试：失败后重试 2 次
- 速率控制：请求间隔 20ms，防止被服务器限流
- 子域名轮换：分散请求到多个服务器

数据持久化（v3.5.3+）：
- 原子写入：先写临时文件再重命名，防止崩溃时数据损坏
- 损坏检测：自动识别损坏的 JSON 文件并恢复为默认值
- 任务恢复：每个下载任务独立持久化，崩溃后自动续传

三、安全特性

- HTTPS 证书验证：默认严格验证，可选关闭（自签名场景）
- 代理支持：所有网络请求可走 HTTP 代理
- 无后台进程：关闭软件后不占用任何系统资源
- 纯本地处理：所有数据处理在本机完成，不上传到云端
- 开源代码：GitHub 上完全公开，可审计

四、跨平台支持

Windows：
- 使用 WebView2（Edge 内核）
- 支持 x64 架构
- 安装包 ~10MB

macOS：
- 使用 WKWebView
- 支持 arm64（Apple Silicon）和 x64（Intel）
- DMG 安装包

Linux：
- 使用 WebKitGTK
- 提供 .deb 和 AppImage 两种分发格式
- 支持 amd64 架构
