# v3.5.0 — 导出性能飞跃 / Wayback 时间过滤 / 瓦片缓存智能跳过

## 关键改进

- **导出直写目标目录**（Issue #32，跳过 TEMP 中转，减少一次完整 IO）
- **跨 strip 并行流水线**（Issue #27，多 strip 并发 + 内存缓冲配置）
- **Wayback 发布日期范围过滤**（扫描时按 releaseDate 筛选）
- **瓦片缓存浏览跳过下载**（浏览过的瓦片自动从队列剔除）
- **启动恢复上次会话状态**（Issue #24）

> 详细分类见下方「性能优化 / 新增 / 修复」三个小节。

## 性能优化

- **原始瓦片直接下载到目标位置（Issue #32）** — 原始瓦片目录导出模式不再经过 TEMP 中转，直接写入最终目标路径，省去一次完整的文件系统拷贝。大批量任务（数万张瓦片）节省数十秒 IO 等待。

- **跨 strip 并行流水线 + 内存缓冲配置（Issue #27）** — streaming_tiff 导出从单 strip 串行升级为多 strip 并发流水线：
  - 下载、解码、编码、写盘四阶段流水并行，充分利用多核 + 磁盘带宽。
  - 新增 `export_buffer_mb` 设置项控制内存缓冲上限，平衡速度与内存占用。

## 新增

- **Wayback 发布日期范围过滤** — 扫描 Wayback releases 时支持按 `releaseDate` 起止日期筛选，避免拉取全量历史版本。配合 always-cache-full 策略，首次扫描后缓存完整 release 列表，后续过滤纯本地计算。

- **瓦片缓存浏览跳过下载（Issue #28）** — 地图浏览过程中已缓存的瓦片自动从下载队列剔除，配合 `active_downloads` 去重机制避免重复请求。

- **启动恢复上次会话状态（Issue #24）** — 应用启动时自动恢复上次关闭前的 tab、图源选择、下载表单参数等状态。

- **Wayback 文件名加 releaseDate** — 避免同一 captureDate 不同 release 的文件互相覆盖。

## 修复

- **切换 tab 后保留下载表单状态（Issue #24）** — 切换页面 tab 不再丢失已填写的下载参数和目录选择。
- **Release notes 解析器分组识别** — 顶部加「关键改进」摘要 + 解析器正确识别小节标题。
