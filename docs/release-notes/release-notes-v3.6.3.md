# v3.6.3 — 修复多边形/圆形裁剪导出"外接矩形 + 黑边"

修复群友反馈的精确裁剪失效：选区为圆形或不规则多边形时，下载出来的却是外接矩形，多边形外是黑边，且黑边里仍含真实影像数据。本版让流式导出真正按形状裁剪，与 v3.2.2 及以下版本的裁剪效果一致。

## 修复

- **多边形/圆形裁剪不再输出外接矩形**
  流式 GeoTIFF / PNG 导出在裁剪时只清除了 alpha 通道，却把整张外接矩形的 RGB 像素全部保留，于是多边形外的每个像素是 `(真实RGB, alpha=0)`。在忽略/拍平 alpha 通道的软件里，这表现为"下载下来的都是外接矩形"，且"黑边里有数据"。现在多边形外的像素整体清零（RGB + alpha），与内存版裁剪 `mask_image_by_polygons` 行为一致——真正按形状裁剪。

  对尊重 alpha 通道的软件（QGIS 等）渲染结果不变；仅修复了被裁掉区域残留底图数据的问题。

- **拆分模式同样精确**：多要素文件按"拆分"导出多个 tif 时，每个要素各自按其多边形精确裁剪，不再是各自的外接矩形。

## 回归时间线

裁剪此前在 v3.2.2 及更早版本走内存路径（`mask_image_by_polygons`，环外 RGB 直接清零）是正确的；v3.3.0 起裁剪改道流式路径，该路径只清 alpha、残留 RGB，引入此问题，并延续到 v3.6.2。

## 验证

新增裁剪回归测试覆盖：
- 合成数据：环外像素必须为 `(0,0,0,0)` 而非 `(RGB,0)`。
- 真实数据：以反馈附带的圆（92 顶点）与不规则多边形（7 顶点）shapefile 几何端到端导出，验证形状内保留影像、外接矩形四角清零。

`cargo test --lib` 66 项全部通过；前端 `tsc -b && vite build` 通过。

## 不影响

下载主流程 / 缓存 / 重试 / 矩形选区裁剪 / DEM 裁剪（本就正确）/ 拆分与批量任务编排逻辑。
